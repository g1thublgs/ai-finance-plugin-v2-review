# 地市场景开发规范

## 总原则

1. 不脱离现有插件样式和框架。
2. 同一个开发包内，三个场景通过目录隔离。
3. 业务逻辑写在本场景目录，不写进路由层。
4. OCR、预填归集、规则审核必须分层清楚。
5. 必须保留并能演示 `other` 其他事项报销场景。
6. 必须提交场景开发文档，不能只交源码。

## 必须开发的文件

| 类型 | 文件 |
|---|---|
| 场景注册 | `domain/scenarios/<scenario>/index.js` |
| OCR 提示词 | `domain/scenarios/<scenario>/ocrPrompt.js` |
| OCR 元数据 | `domain/scenarios/<scenario>/ocrProfile.js` |
| 预填/归集 | `domain/scenarios/<scenario>/prefillModel.js` 或 `aggregation/` |
| 规则模型 | `domain/scenarios/<scenario>/ruleModel.js` |
| 表单字段 | `domain/scenarios/<scenario>/formSchema.js` |
| Python 规则 | `services/rules/python/<scenario>/rule_*.py` |

## 禁止事项

1. 不修改其他地市负责的场景目录。
2. 不恢复或复制差旅费核心目录。
3. 不把规则判断写在插件前端。
4. 不把多条指标塞进一个规则文件。
5. 不提交真实人员、金额、票据等敏感测试数据。
