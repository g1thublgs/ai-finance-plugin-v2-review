## 三、字段模型说明

OCR 输出材料类型：

- `meetingNotice`、`meetingApproval`、`meetingPlan`、`attendanceList`、`meetingSettlement`、`accommodationList`、`normalInvoice`、`paymentProof`、`other`

预填归集字段：

- `records`：每份 OCR 材料形成一条可追溯记录。
- `summary`：会议名称、起止日期、地点、天数、人数、材料布尔值、住宿费、伙食费、场地租金、资料费、交通费、其他费用、发票金额、支付金额、申请金额、合计金额。
- `evidence`：字段来源和值。
- `warnings`：字段冲突、缺失、重复发票等归集提示。

页面字段映射：

- `SQ_MC` -> `title`/`meetingName`
- `SQ_SY` -> `reason`
- `JBR_MC`/`SQ_JBR` -> `applicantName`
- `SSBM_MC`/`SQ_SSBM` -> `departmentName`
- `HYTS` -> `meetingDays`
- `HYRS` -> `attendeeCount`
- `HSF` -> `mealAmount`
- `ZSF` -> `accommodationAmount`
- `CDF` -> `venueRentAmount`
- `QTFY` -> `otherAmount`
- `SQ_JE` -> `applyAmount`/`totalAmount`
- `PJZS`、`HYPXBH`、`HYJYBH`、`BZ` 保留为附件张数、计划序号、批件名称、备注。

字段来源优先级：

- 人数：页面字段 -> 预填汇总 -> 签到名单数量 -> OCR count。
- 天数：页面字段 -> 预填汇总 -> 起止日期计算。
- 金额：页面字段优先用于申请信息类规则；发票金额按发票号码或销售方/日期/金额去重。
- 材料完整性：OCR `recognizeType` 和预填汇总布尔字段共同判断。

## 四、主要改动说明

- `ocrPrompt.js`：从占位模板扩展为会议费专用结构化 JSON 数组模板，覆盖通知、审批、计划、签到、结算、住宿、发票、付款凭证。
- `ocrProfile.js`：同步材料类型和会议费关键词，避免加入“出差、培训、接待”等宽泛关键词。
- `formSchema.js`：扩展会议费字段模型，兼容原有字段并新增规则所需字段。
- `prefillModel.js`：重构为会议费归集器，提供文本、金额、数字、日期、候选字段、数组和 OCR 兜底读取函数。
- `ruleModel.js`：按规则清单拆成 14 条元数据，关联 `rule_*.py`。
- `meeting_common.py`：封装会议费规则共用容错函数、返回结构、会议类别/人数/天数/发票金额读取。
- `rule_*.py`：一条规则一个文件，字段缺失时返回人工复核提示，不抛异常。
- 前端：`popup.js` 增加会议费后端归集/审核调用和展示；`content.js` 增加会议费页面字段提取。

## 五、稳定性与容错设计

- 空字段：规则通过 `make_skip` 或 warning 提示人工复核。
- 金额解析：兼容人民币符号、逗号、中文“元”、空字符串。
- 日期解析：兼容 `YYYY-MM-DD`、`YYYY年M月D日`、斜杠和点号。
- 人数解析：优先页面字段，缺失时读签到表名单或 count。
- 材料缺失：按规则清单要求提示，不直接导致引擎失败。
- OCR 失败：保留 `rawText`、`ocrItems`，无结构字段时规则提示人工复核。
- 页面/OCR 冲突：归集阶段写入 `warnings` 和 `evidence`，不静默覆盖。
- 规则异常防护：规则文件均使用会议费本地公共函数，测试中 14 条规则无 error。
